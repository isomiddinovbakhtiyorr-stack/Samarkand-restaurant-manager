# Samarkand Restaurant Manager

Полностью офлайн PWA-приложение для владельца ресторана: продажи, расходы, касса, склад, отчёты. Один пользователь, вход по PIN-коду, все данные хранятся локально в IndexedDB.

## Запуск

```bash
npm install
npm run dev
```

Откройте адрес, который выведет Vite (обычно `http://localhost:5173`). При первом запуске приложение покажет экран настройки: язык, название ресторана, PIN-код.

## Сборка production-версии

```bash
npm run build
npm run preview
```

Папку `dist/` можно разместить на любом статическом хостинге (или открыть локально) — приложение полностью самодостаточно и не обращается к серверу.

## Установка как PWA на Android

1. Откройте приложение в Chrome на Android (после `npm run build` разместите `dist/` на любом https-хостинге, либо на локальном сервере в той же Wi-Fi сети телефона и ноутбука).
2. Chrome предложит «Добавить на главный экран» / «Установить приложение».
3. После установки приложение работает как обычное Android-приложение, полностью офлайн.

## Структура проекта

```
src/
  db/            IndexedDB: обёртка + репозитории (sales, expenses, inventory, cash, settings)
  context/       AppContext — язык, настройки, авторизация по PIN
  i18n/          Словари переводов ru / en / uz
  components/    Переиспользуемые UI-компоненты (TopBar, Toast, ProtectedRoute)
  screens/       Экраны: Setup, Login, Dashboard, Sales, Expenses, Cash, Inventory, Reports, Settings
  styles/        global.css — чёрный фон, белые карточки, жёлтые кнопки
  utils/         Даты, форматирование валюты
```

## База данных

Используется нативный IndexedDB (без внешних библиотек) — база `SamarkandRestaurantDB`, таблицы:

- `sales` — записи продаж по дате (cash / card / online / total)
- `expenses` — расходы (категория, поставщик, сумма, комментарий)
- `inventory` — текущие остатки по продуктам
- `cash` — кассовые дни (opening / cash sales / cash expenses / expected / actual / difference)
- `settings` — название ресторана, PIN, валюта, язык

Резервное копирование и восстановление — в разделе «Настройки» (экспорт/импорт JSON-файла со всеми таблицами).

## Перенос на Flutter

Слой `src/db/*Repo.js` уже изолирует всю бизнes-логику от UI: суммы, закрытие дня, отчёты считаются в репозиториях, а не в компонентах. При переносе на Flutter эти функции переносятся на Dart почти дословно (SQLite/Hive вместо IndexedDB), а экраны переписываются как Flutter-виджеты по тем же данным и состояниям.

## Технологии

Vite, React, React Router (HashRouter — работает офлайн без серверной маршрутизации), vite-plugin-pwa, нативный IndexedDB. Без Firebase и без серверной части — вся логика работает локально на устройстве.
